-- OOP'S + Design Patterns

    - Q2. Why use Generics?
    👉 For reusable and type-safe code.
    - Q3. Difference between List and Map?
    👉 List stores ordered items, Map stores key-value pairs.

-- OOP'S + Design Patterns + Async Programming
    - Error & Exception Handling
    ANS) Exception handling prevents app crashes and handles unexpected errors safely.

    | Exception        | Meaning          |
    | ---------------- | ---------------- |
    | FormatException  | Invalid format   |
    | SocketException  | Internet issue   |
    | TimeoutException | API timeout      |
    | Null Exception   | Null value issue |

    - Memory & Performance Optimization
    ANS) Optimization improves app speed and reduces memory usage.

    | Optimization        | Use                    |
    | ------------------- | ---------------------- |
    | const widgets       | Prevent rebuild        |
    | ListView.builder    | Lazy loading           |
    | Dispose controllers | Prevent memory leak    |
    | Isolates            | Heavy background work  |
    | Pagination          | Reduce large data load |

    - Packages & Libraries
    ANS) Packages provide reusable functionality.
    | Package            | Use                  |
    | ------------------ | -------------------- |
    | dio                | API                  |
    | flutter_bloc       | State management     |
    | get_it             | Dependency injection |
    | hive               | Local database       |
    | shared_preferences | Small local storage  |

    - Packages & Libraries Management
    ANS) Managing versions and dependencies correctly.
    
    - Asynchronous Basics
    ANS) Async programming executes tasks without blocking UI.
    | Keyword | Meaning         |
    | ------- | --------------- |
    | async   | Async function  |
    | await   | Wait for result |
    | Future  | Future result   |

    - Asynchronous Programming
    ANS) Multiple tasks run independently without blocking main thread.

    - Async State Handling
    ANS) Managing loading, success, and error states.

    - Streams & Reactive Programming
    ANS) Streams continuously emit data over time.

    - Isolates & Multithreading
    ANS) Isolates run heavy tasks in background.
